Installing Phosphorus.Five
  1. Make sure you've got MonoDevelop, Xamarin or Visual Studio on your system. If you use MonoDevelop, you'll have to explicitly install xsp in addition, which is a small web-server allowing you to run ASP.NET applications. On Ubuntu, you can use Ubuntu Software Center, and search for "MonoDevelop" and "xsp 4.0". On Windows, all you need is Visual Studio, alternatively Xamarin. On Mac OSX, Xamarin should be sufficient.
  2. Then make sure you download the latest stable release of Phosphorus.Five, and extract it on your system.
  3. Open "phosphorus.sln" in either Visual Studio, Xamarin or MonoDevelop.
  4. Choose "Start without Debugging" from the Run or Debug menu.

If you wish to use System42 together with Phosphorus.Five, you must remember to edit your "web.config" file, and make sure it says "system42/application-startup.hl" as the value of your "application-startup-file" appSettings key. And that you have downloaded and extracted System42 into your "core/phosphorus.application-startup" folder.

Conventions used in documentation.

All example code is written in a code box, with a blue line to its left, having an "x" in its top left corner, like the below example;

foo
  bar

When the documentation is referring to nodes, and/or Active Events, it will often put the names of the nodes, and/or Active Events, inside square brackets, and make the reference become bold. For instance, to refer to the above "foo" node, would normally be something like this; [foo]

Most example code in this documentation, is written such that it will produce some sort of output. A good advice to see its output, is to run the example code through the "pf.lambda executor" in System42, which allows you to see the output of executed pf.lambda code.

PS!
Even though this is the documentation for the C# parts of the system, it also provides extensive documentation for the pure "pf.lambda" user, who exclusively wishes to use the pf.lambda parts of Phosphorus.Five, which can be seen through classes such as the Expression class, providing extensive documentation about the pf.lambda expressions for the library.

Also the Iterators are highly useful for the pure pf.lambda user to read up about, since they document every type of pf.lambda iterator you can use when composing your pf.lambda expressions. In addition to of course the pf.lambda keywords, which for the most parts can be found in the keywords namespace.

All Active Events are also extensively documented, and can be found through the "Modules" link at the top of this page.
