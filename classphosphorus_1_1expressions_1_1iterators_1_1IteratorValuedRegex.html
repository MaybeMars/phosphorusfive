Detailed Description

Returns all nodes having a value matching the given regular expression.

Will return all nodes who's value matches the given regular expression. Notice that when you use this Iterator, the entire iterator's value must be put into either double quote ("), to signify a single line string literal, or created using multi-line strings, since the iterator is having "slash" (/) as part of its control mechanisms.

Example; 

/"=/foo/"

Will match all nodes who's value contains the text "foo".

You can optionally pass in arguments to the regular expression engine, after the last slash (/) of your iterator's content. These parameters can be either of;

d - Only return distinct names. If two nodes have the same names, only return the first node matching.

i - Ignore case when doing the comparison.

m - Multiline regex comparison.

c - Compile the regular expression to optimize execution for later usage of the same regular expression.

e - Use ECMA script regular expression type.

w - Ignore white space patterns.

r - Execute the regular expression from right to left.

s - Single line regular expression mode.

Here's an example of how to use ECMA script type of regular expressions, compile the regular expression, while only returning distinct values, to return any node's who's values contains the text "bar";

/"=/bar/edc"
