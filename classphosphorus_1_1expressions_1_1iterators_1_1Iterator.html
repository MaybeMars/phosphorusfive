Detailed Description

Iterator class, wrapping one iterator, for pf.lambda expressions.

Iterators is at the heart of Expressions, and are executed as a chain, encapsulated within a Logical object, which again is a child of an IteratorGroup. When iterators are evaluated, they are evaluated in a left-associative manner, where the last iterator is evaluated last, and evaluated as a result of its previous iterator's Left property.

The root iterator of all expressions, is normally an iterator, containing a single Node instance, from which iteration begins. Below is an example of an Expression containing two iterators;

@/../*?node

The above Expression has one IteratorRoot iterator, and one IteratorChildren iterator, and has the 'value' type declaration, which means it will extract the 'value' of its resulting nodes. Effectively extracting all value properties, from all children nodes, of the root node of your expression's execution tree.

virtual bool phosphorus.expressions.iterators.Iterator.IsReference
getset

Returns true if this is a "reference expression".

A reference expression is normally defined by having two consecutive @ characters in its beginning. This signals to the Expression engine that the expression is not necessarily to be directly evaluated, but that it might point to another expression, and if it does, then that Expression will be evaluated, and the results of that evaluation process returned back to caller.

true if this is a reference expression, otherwise false.
